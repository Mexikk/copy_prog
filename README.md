# Copy_prog

Утилита для сбора файлов из монорепозитория в единый документ с заголовками путей. Поддерживает мультиязычность (RU/EN/ZH), превью списка файлов, выбор конкретных директорий для обработки, гибкие исключения и интерактивный режим.

## Возможности
- **Выбор конкретных папок**: Сканирование и копирование файлов только из указанных подпапок (оптимизация обхода дерева).
- **Превью (Preview)**: Оценка списка файлов до копирования с пагинацией и маркерами статусов (`[OK]`, `[ИСКЛ]`, `[СИСТ]`, `[РАСШ]`, `[НЕ ВКЛ]`).
- **Мультиязычность**: Интерфейс на русском, английском и китайском языках.
- **Автоматическая настройка Unicode**: Программа автоматически настраивает консоль Windows для корректного отображения русского, английского и китайского языков (UTF-8).
- **Надёжная фильтрация путей**: Использование канонических абсолютных путей гарантирует корректную работу независимо от формата ввода (относительные/абсолютные пути, с слешами или без).
- **Безопасность**: Строгая проверка UTF-8, защита от выхода за пределы директории.
- **Производительность**: Потоковый обход дерева, пропуск системных и "невключённых" папок без глубокого сканирования.
- **Кроссплатформенность**: Полная поддержка Windows, Linux и macOS.

## 🛠️ Установка для разработчиков

### Через Cargo (рекомендуется, если есть Rust)

Если у вас установлен [Rust](https://www.rust-lang.org/tools/install):

```bash
cargo install --git https://github.com/Mexikk/copy-prog.git
```

Программа автоматически установится в `~/.cargo/bin/` и будет доступна из любого места по команде `copy_prog`.

### Из исходного кода

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Mexikk/copy-prog.git
   cd copy-prog
   ```

2. Соберите программу:
   ```bash
   cargo build --release
   ```

3. Запустите:
   - **Windows:** `.\target\release\copy_prog.exe`
   - **Linux/macOS:** `./target/release/copy_prog`

4. Опционально установите в систему:
   ```bash
   cargo install --path .
   ```

### Зависимости для сборки

**Windows:** Достаточно установить Rust. Windows API подключается автоматически.

**Ubuntu/Debian:**
```bash
sudo apt install build-essential pkg-config
```

**Fedora:**
```bash
sudo dnf groupinstall "Development Tools"
sudo dnf install pkgconfig
```

**Arch Linux:**
```bash
sudo pacman -S base-devel pkgconf
```

**macOS:**
```bash
xcode-select --install
```

## 🌍 Требования для корректного отображения текста

### Windows

**Рекомендуется:** Используйте [Windows Terminal](https://aka.ms/terminal) из Microsoft Store — он поддерживает все символы Unicode (кириллицу, латиницу, иероглифы) из коробки.

**Альтернатива (cmd.exe/PowerShell):**
Программа автоматически настраивает консоль на UTF-8 при запуске. Однако для корректного отображения всех символов может потребоваться:

1. **Для русского языка (кириллицы):**
   - Если у вас англоязычная Windows, установите русский языковой пакет:
     Параметры → Время и язык → Язык → Добавить язык → Русский
   - Шрифт консоли: **Consolas**, **Lucida Console** или **Courier New** (поддерживают кириллицу)

2. **Для китайского языка (иероглифов):**
   - Установите китайский языковой пакет:
     Параметры → Время и язык → Язык → Добавить язык → 中文(简体)
   - Шрифт консоли: **Microsoft YaHei**, **SimSun-ExtB** или **NSimSun**
   - ⚠️ Стандартные шрифты cmd.exe часто не поддерживают иероглифы — используйте Windows Terminal

3. **Для английского языка:**
   - Отображается корректно на любой системе без дополнительной настройки

### Linux/macOS

Современные терминалы (GNOME Terminal, Konsole, iTerm2, Terminal.app) поддерживают UTF-8 по умолчанию. Убедитесь, что установлен шрифт с поддержкой CJK (например, **Noto Sans CJK** или **Source Code Pro**).

**Для китайских иероглифов:**
```bash
# Ubuntu/Debian
sudo apt install fonts-noto-cjk

# Fedora
sudo dnf install google-noto-sans-cjk-fonts

# Arch
sudo pacman -S noto-fonts-cjk
```

### Устранение неполадок

Если вместо текста вы видите `????` или `□□□`:

1. Запустите программу через **Windows Terminal** (рекомендуется)
2. В cmd.exe/PowerShell измените шрифт: правый клик по заголовку → Свойства → Шрифт
3. Убедитесь, что установлен нужный языковой пакет Windows
4. Для старых версий Windows (7/8) может потребоваться обновление Unicode-поддержки

## 📋 Интерактивное меню
1. Выбрать исходную папку.
2. Выбрать определённые папки.
3. Настроить расширения (добавить/удалить/режим «все файлы»).
4. Настроить разделитель (0–5 пустых строк).
5. Включить/выключить игнорирование служебных папок.
6. Настроить исключения по путям (добавить/удалить/показать/очистить).
7. Выбрать выходной файл.
8. Запустить копирование.
9. Показать превью списка файлов (что будет скопировано).
10. Сбросить настройки до значений по умолчанию.
11. Сохранить текущие настройки как «по умолчанию».
12. Показать встроенную инструкцию.
13. Переключить язык интерфейса ([Рус] [Eng] [中文]).
14. Выход.

## 💻 Примеры использования

### Интерактивный режим (по умолчанию)
```bash
copy_prog
```

### CLI режимы

1. **Копирование с 0 разделителями и только `.rs`/`.md`**
   ```bash
   copy_prog --source . --sep 0 --exts .rs,.md
   ```

2. **Включение системных папок (сканировать `target`, `.git` и т.д.)**
   ```bash
   copy_prog --source . --include-system-dirs
   ```

3. **Добавление исключений (пропустить `target` и конкретный файл `shared/src/legacy.rs`)**
   ```bash
   copy_prog --source . --exclude target,shared/src/legacy.rs
   ```

4. **Запуск с английским интерфейсом и превью (без реального копирования)**
   ```bash
   copy_prog --lang EN --preview
   ```

5. **Копирование только из конкретных папок (`src` и `shared`)**
   ```bash
   copy_prog --source . --only-dirs src,shared
   ```

6. **Сброс настроек до заводских**
   ```bash
   copy_prog --reset-config
   ```

7. **Показать расширенную инструкцию**
   ```bash
   copy_prog --usage-details
   ```
